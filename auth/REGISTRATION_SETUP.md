# রেজিস্ট্রেশন সিস্টেম - দ্রুত সেটআপ

## সমস্যা
রেজিস্ট্রেশন ফর্ম সাবমিট করলে কোন বার্তা দেখা যায় না এবং লগইন পেজে যায় না।

## সমাধান

### ধাপ 1: ডাটাবেস সেটআপ
আপনার ডাটাবেসে নিম্নলিখিত কলামগুলি নিশ্চিত করুন:

```sql
-- আপনার users টেবিলে এই কলামগুলি থাকা উচিত:
ALTER TABLE users ADD COLUMN IF NOT EXISTS is_verified TINYINT(1) DEFAULT 0;
ALTER TABLE users ADD COLUMN IF NOT EXISTS is_active TINYINT(1) DEFAULT 1;
ALTER TABLE users ADD COLUMN IF NOT EXISTS login_attempts INT DEFAULT 0;
```

### ধাপ 2: টেস্ট করুন
এই URL এ যান এবং আপনার সিস্টেম টেস্ট করুন:

```
http://localhost/mental%20health/auth/test-register.php
```

এটি দেখাবে:
- ✓ PHP চলছে কি না
- ✓ ডাটাবেস সুসংযুক্ত কি না
- ✓ প্রয়োজনীয় টেবিল আছে কি না

### ধাপ 3: সিস্টেম সেটআপ

**যদি সব সবুজ হয়:**
- রেজিস্টার পেজে যান: http://localhost/mental%20health/auth/register.php
- ফর্ম পূরণ করুন
- সাবমিট করুন
- সফলতার বার্তা দেখা যাবে এবং লগইন পেজে যাবে

**যদি কোন ত্রুটি দেখায়:**
- ত্রুটির বার্তা পড়ুন এবং সেই অনুযায়ী ডাটাবেস সেটআপ করুন
- test-register.php এর FORM সেকশন ব্যবহার করে ডাইরেক্ট ফর্ম জমা দিন

### ধাপ 4: ডাটাবেস দেখুন
phpMyAdmin এ যান এবং নতুন ইউজার যোগ হয়েছে কি না তা চেক করুন:

```
http://localhost/phpmyadmin
যান -> mentora_db -> users টেবিল
```

## সিস্টেম কীভাবে কাজ করে

1. **ফর্ম পূরণ** → ইউজার সমস্ত ফিল্ড পূরণ করে এবং সাবমিট করে
2. **ক্লায়েন্ট ভ্যালিডেশন** → JavaScript চেক করে সব ফিল্ড সঠিক আছে
3. **AJAX রিকোয়েস্ট** → ফর্ম ডেটা register-process.php এ পাঠায়
4. **সার্ভার ভ্যালিডেশন** → সার্ভার দ্বিতীয় বার চেক করে
5. **ডাটাবেস সংরক্ষণ** → ডেটা users টেবিলে সংরক্ষিত হয়
6. **সফলতা প্রতিক্রিয়া** → JSON প্রতিক্রিয়া পাঠায়
7. **রিডাইরেক্ট** → ইউজার লগইন পেজে রিডাইরেক্ট হয়

## ব্রাউজার কনসোল চেক করা

যদি কিছু কাজ না করে:

1. ব্রাউজার খুলুন (F12 বা ডান ক্লিক > Inspect)
2. **Console** ট্যাব খুলুন
3. রেজিস্টার ফর্ম সাবমিট করুন
4. console.log মেসেজগুলি দেখুন - এটি সমস্যা বলে দেবে

| সমস্যা | কারণ | সমাধান |
